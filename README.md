# Лабораторная работа №11
Срок сдачи работы: 2026-03-21.

Оценка выставляется в 10 балльной шкале.
После истечения срока работа считается не сданной и оценивается в 0 баллов.

Все работы проверяются на отсутствие ошибок сборки в автоматическом режиме,
в случае отсутствия ошибок выполняется ручная проверка кода, в противном
случае работа оценивается в 0 баллов.

В ходе ручной проверки работ оценивается полнота выполнения задания,
а также качество решения.

## DevOps. Непрерывная интеграция. Конвейеры

### Ответвление
Необходимо создать ответвление (fork) моего репозитория Lab-11 (название
оставить без изменения, а видимость поставить приватной). Назначить меня в
новом репозитории соавтором с правами администратора. Вики, Задачи а также
Проекты отключить.

В локальной копии создавать ветки (branch) по необходимости.

### Этап разработки
Разработать программу на языке Си++ (ISO/IEC 14882:2017), демонстрирующую
решение поставленной задачи. Основной код разместить в заголовочном файле
`Vector.hpp`, тесты в `test.cpp`. По выполненной работе составить отчет
согласно требованиям ГОСТ 7.32-2017 содержащий диаграмму классов согласно
спецификации UML 2.0 и предоставить его копию в формате PDF. Все необходимые
файлы разместить в git-репозитории.

1. Доработать шаблон класса `Vector` (из ЛР-9) добавив функционал связанный
с семантикой перемещения.

2. Программа должна содержать файл конфигурации системы CMake.

3. Для тестов использовать библиотеку GoogleTest.
Сборка и запуск тестов должны быть опциональными и управляться CMake.

4. Подключить репозиторий к системе Woodpecker CI,
конфигурацию разместить в файле `.woodpecker.yaml`.

5. Обеспечить проверку отсутствия ошибок сборки посредством Woodpecker CI.
Компилятор и операционная система, используемые при проверке:
 GNU Compiler Collection 15.2.0 цель x86_64-slackware-linux
 система Slackware Linux current 2025-09-01.
Флаги компиляции: `-Wall` `-Werror` `-pedantic`.
В случае отсутствия ошибок сборки автоматически открыть запрос на слияние (читай плугины).

6. Обеспечить выполнение проверки корректности работы посредством Woodpecker CI
(запустить тесты; компиляция с опцией `-fsanitize`).
Обеспечить выполнение анализа покрытия кода посредством Woodpecker CI (см. справку).
Обеспечить создание комментария с отчетом о покрытии в запросе на слияние посредством Woodpecker CI (читай плугины).

7. Обеспечить выполнение статического анализа кода (cppcheck/clang-tidy) посредством Woodpecker CI (см. справку).
Обеспечить создание комментария с отчетом о результате в запросе на слияние посредством Woodpecker CI (читай плугины).

### Этап обсуждения
По результатам рецензирования получить минимум один комментарий,
в случае необходимости внести исправления.
